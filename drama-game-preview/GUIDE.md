# 剧情织造机 V1.3.19 - 游戏说明与游玩指引

## 📖 游戏简介

**剧情织造机 (Text Game Weaver)** 是一款交互式剧情游戏编辑器，让您能够轻松创建、编辑和体验丰富的文字冒险故事。无论是简单的选择剧情还是复杂的分支叙事，都能在这里实现。

### 🎮 核心特色

- **双模式生成**：快速模式一键生成，沉浸模式逐步选择
- **智能匹配系统**：基于标签自动匹配合适的玩法和场景
- **丰富的元素库**：场景、层级、玩法、指令四大元素构建完整故事
- **本地数据持久化**：所有作品自动保存，永不丢失
- **导出分享**：支持多种格式导出，方便分享和备份

## 🚀 快速开始

### 第一步：创建基础元素

#### 1. 场景设置
场景是故事发生的具体地点，每个场景都有独特的氛围和标签。

**操作路径**：底部导航 → 场景

**示例场景**：
```
名称：废弃实验室
描述：充满灰尘和化学试剂味道的房间，墙角散落着破损的实验器材。
标签：恐怖, 解谜, 室内
```

#### 2. 层级设计
层级定义了故事的发展顺序，从开场到结局的完整流程。

**操作路径**：底部导航 → 层级

**默认层级**：
- 第一幕：开场准备 (L1_SETUP)
- 第二幕：核心解谜 (L2_CORE)  
- 第三幕：结局判定 (L3_ENDING)

#### 3. 玩法创作
玩法是玩家在各个层级中可以执行的具体行动。

**操作路径**：底部导航 → 玩法

**示例玩法**：
```
名称：搜寻物资
描述：在角落里翻找，发现了一个急救包。
触发条件：无
预期结果：获得急救包
所属层级：开场准备
标签：生存
```

#### 4. 指令配置
指令是随机触发的特殊事件，增加故事的不可预测性。

**操作路径**：底部导航 → 指令

**示例指令**：
```
名称：突发地震
描述：地面剧烈摇晃，物品掉落。
触发概率：30%
作用域：特定层级
目标ID：L2_CORE
```

## 🎯 两种游戏模式

### 📝 快速模式（一键生成）

适合快速体验和预览故事流程。

**使用步骤**：
1. 点击底部"生成"按钮
2. 选择起始场景
3. 点击"一键生成流程"
4. 系统自动生成完整故事
5. 查看结果并可保存到历史记录

**特点**：
- 自动选择匹配的玩法
- 随机触发符合条件的指令
- 一次性生成完整故事线

### 🎭 沉浸模式（剧情走向）

适合深度体验和互动式剧情创作。

**使用步骤**：
1. 选择起始场景
2. 点击"开启剧情走向"
3. 逐层选择想要的玩法和指令
4. 查看每层的故事发展
5. 完成后自动保存到历史记录

**特点**：
- 逐层推进，完全控制故事走向
- 智能推荐3-4个最匹配的玩法选项
- 支持多选决策，组合不同玩法
- 实时反馈，每选择都能看到故事进展

## 💡 进阶技巧

### 🏷️ 标签系统

标签是智能匹配的核心，合理使用标签可以提升故事质量：

**场景标签**：描述场景特征
- 环境：室内, 室外, 森林, 城市, 洞穴
- 时间：日间, 夜晚, 黎明, 黄昏
- 氛围：恐怖, 温馨, 神秘, 紧张

**玩法标签**：描述玩法特征  
- 类型：解谜, 战斗, 探索, 社交, 生存
- 难度：简单, 中等, 困难
- 互动：单人, 多人, NPC

### 🔧 智能匹配机制

系统会根据标签匹配度智能推荐玩法：

1. **完美匹配**：标签完全匹配的玩法优先级最高
2. **部分匹配**：有共同标签的玩法次优先级
3. **随机选择**：同优先级玩法中随机选择

### 🎲 概率控制

指令的触发概率影响故事体验：

- **低概率 (1-20%)**：罕见事件，增加惊喜感
- **中概率 (21-50%)**：常见事件，丰富故事内容  
- **高概率 (51-100%)**：频繁事件，可能改变故事走向

## 📚 数据管理

### 💾 自动保存

所有数据都会自动保存到浏览器本地存储：

- ✅ 场景、层级、玩法、指令
- ✅ 生成历史记录
- ✅ 用户界面状态
- ✅ 当前编辑内容

**保存时机**：
- 数据变化时（500ms防抖）
- 页面隐藏时
- 页面卸载时
- 每30秒定期保存

### 🗂️ 历史记录

所有生成的故事都会保存到历史记录中：

**功能**：
- 查看历史故事列表
- 重新加载历史故事
- 导出为TXT文件
- 复制故事文本
- 编辑故事标题
- 删除不需要的记录

### 📤 导入导出

支持多种数据管理方式：

#### 1. JSON格式
- **导出**：完整数据备份，包含所有元素
- **导入**：从JSON文件恢复数据

#### 2. TXT格式  
- **单个故事**：导出单个故事的可读文本
- **批量导出**：一次性导出所有历史故事

#### 3. 分享链接
- 生成Data URI链接，方便分享配置
- 支持跨设备数据传输

## 🚨 常见问题

### Q: 数据会丢失吗？
A: 不会！所有数据都自动保存到浏览器本地存储，页面刷新不会丢失。

### Q: 如何备份我的作品？
A: 在设置页面可以导出JSON文件进行备份，或者使用分享链接功能。

### Q: 标签怎么设置最合理？
A: 建议使用环境、类型、难度等维度进行分类，这样能让系统更准确地匹配。

### Q: 概率设置多少合适？
A: 根据指令重要性设置，重要事件概率低一些，普通事件概率高一些。

### Q: 可以创建多结局故事吗？
A: 可以！通过在不同层级设置不同的玩法组合，可以创建分支剧情。

### Q: 如何分享我的故事？
A: 生成后可以在历史记录中导出TXT文件分享，或者使用复制文本功能。

## 🎮 创作建议

### 📖 故事结构

1. **开场铺垫**：建立场景氛围，引出核心矛盾
2. **发展探索**：通过多个玩法推进剧情
3. **高潮转折**：使用指令制造意外和紧张感
4. **结局收尾**：根据玩家选择给出相应结果

### 🎭 氛围营造

- **恐怖故事**：使用恐怖标签的场景，配合突发指令
- **解谜故事**：重点设计解谜类玩法，降低干扰指令概率  
- **冒险故事**：平衡探索和战斗玩法，增加随机事件
- **温馨故事**：使用温馨场景，设计社交类玩法

### 🔁 循环利用

- 创建高质量的场景模板，方便复用
- 建立标准化的玩法库，提高创作效率
- 设置合理的指令库，增强故事多样性
